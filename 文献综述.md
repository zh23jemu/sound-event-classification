# 基于 Mel-Spectrogram 与 Transformer 的声音事件分类研究文献综述

## 摘要

声音事件分类是计算机听觉中的基础任务，目标是从环境音频中识别具有语义意义的声音类别。已有研究经历了从手工声学特征、卷积神经网络，到频谱图 Transformer 和自监督预训练模型的发展过程。早期方法依赖 MFCC、谱能量等人工特征，表达能力受限；基于 Log-Mel Spectrogram 的 CNN 方法提升了局部时频模式建模能力，但对长程时间依赖和跨频带关系的表达仍有限。近年来，Vision Transformer 将二维输入切分为 patch 并使用自注意力建模全局关系，为音频频谱图分类提供了新的结构基础；Audio Spectrogram Transformer、PaSST 和 HTS-AT 等工作进一步证明，频谱图 Transformer 能够在音频分类、声音事件检测和音频标注任务中取得较强表现。同时，PANNs、SSAST、Audio-MAE、BYOL-A 等研究表明，大规模预训练和自监督学习可以缓解标注数据不足问题；SpecAugment、mixup、Patchout 和 PSLA 等训练策略则说明，数据增强、采样和标签处理对模型泛化同样重要。本文围绕声音事件分类、音频 Transformer、自监督音频表征、音视频学习和数据增强五个主题梳理已有研究，比较不同方法的优势、局限和发展趋势，并总结该方向仍存在的小样本泛化、多标签噪声、频谱图归纳偏置和多模态迁移成本等研究空白。

**关键词**：声音事件分类；Log-Mel Spectrogram；Audio Spectrogram Transformer；自监督学习；音视频学习；数据增强

## 1. 引言

声音事件分类关注非语音或广义音频中的语义事件识别，例如交通声、自然声、人类活动声、动物声和机械声等。与语音识别主要分析语言内容不同，声音事件分类更强调环境声的时频结构、背景噪声、类别共现和弱标签问题。Piczak 构建的 ESC-50 数据集推动了环境声音分类基准研究，使研究者能够在统一类别体系上比较传统特征、CNN 和后续深度模型[2]。AudioSet 进一步把任务扩展到大规模、多标签和弱标注场景，其声音事件本体和 YouTube 片段标注使音频事件识别进入大规模预训练阶段[4]。FSD50K 则基于 Freesound 音频和 AudioSet Ontology 提供人工标注数据，为真实声音事件分类、多标签学习和类别不均衡研究提供了更接近实际应用的基准[5]。

从方法演进看，声音事件分类最初常使用 MFCC、谱质心、短时能量、过零率等人工特征，并与支持向量机或浅层模型结合。这类方法计算成本较低，但难以表达复杂环境声中的非线性时频结构。随着深度学习发展，研究逐渐转向将音频转换为 Mel-Spectrogram 或 Log-Mel Spectrogram，再使用 CNN 学习局部时频纹理[3]。CNN 模型适合捕捉局部模式，但环境声音常具有长时间结构和跨频带依赖，例如警笛声的周期性变化、交通声的持续背景和人群声的多源叠加。Transformer 的自注意力机制能够直接建模任意 token 之间的关系，因此被引入音频频谱图建模[1][7]。

本文的综述重点是已有研究脉络，而不是项目实施方案。下文将按照“任务与数据集、时频表示与 CNN、音频 Transformer、自监督预训练、数据增强与训练策略、音视频学习、综合分析”展开，说明不同研究如何共同推动声音事件分类从局部特征学习走向全局建模和预训练迁移。

## 2. 声音事件分类任务与数据集研究

声音事件分类既可以是单标签任务，也可以是多标签任务。ESC-50 这类环境声音数据集通常假定一段音频对应一个主类别，适合评估基础分类能力[2]；AudioSet 和 FSD50K 更接近真实环境，一段音频可能同时包含多个声音事件，并且标签常常是 clip-level 的弱标签[4][5]。这种差异决定了评价指标也不同：单标签任务通常使用 Accuracy；多标签任务更常使用 mAP、micro-F1、macro-F1 等指标来衡量整体排序能力、频繁类别表现和长尾类别表现[5][18]。

数据集研究还揭示了声音事件分类的三个核心难点。第一，环境音频的时间尺度差异很大，玻璃破碎等瞬态声与雨声、发动机声等持续声需要不同的时间建模能力。第二，类别边界并不总是清晰，流水声、雨声和风声可能在频谱纹理上相似。第三，大规模数据常伴随弱标签、标签噪声和类别不均衡。AudioSet 的大规模本体结构为预训练提供了丰富语义覆盖，但其 YouTube 来源也带来弱标注和样本可用性问题[4]。FSD50K 通过人工标注提高了质量，但仍保留多标签、不平衡和音频长度不一致等真实挑战[5]。

因此，数据集并不只是实验材料，而是决定模型设计和评价方式的重要因素。小规模单标签数据集便于快速比较方法，但容易高估大型模型的实际泛化能力；大规模多标签数据集更贴近真实应用，却要求模型处理标签噪声、类别共现和复杂评价指标[4][5][18]。这一点是理解后续模型发展的基础。

## 3. 时频表示、CNN 与预训练音频网络

Log-Mel Spectrogram 是深度音频分类中最常见的输入形式之一。它把一维波形转换为二维时频能量图，横轴对应时间，纵轴对应 Mel 频率，既保留了事件随时间变化的信息，又贴近人耳频率感知。Piczak 的环境声音 CNN 研究展示了将频谱图视为二维输入并使用卷积提取局部声学模式的有效性[3]。与传统人工特征相比，CNN 可以自动学习短促高频、持续低频、周期纹理等局部结构，因此在环境声、声景分类和音频标注任务中长期作为强基线。

CNN 方法的优势是局部归纳偏置强、训练相对稳定，并且在样本量不大时通常比完全无归纳偏置的模型更容易优化。PANNs 进一步把 CNN 扩展到大规模预训练范式，通过在 AudioSet 上训练多种 CNN 架构，再迁移到声学场景分类、音乐分类和声音事件相关任务，说明大规模音频预训练可以提高下游任务表现[6]。这类研究的重要意义在于，它将“模型结构”问题和“预训练数据”问题结合起来，证明音频分类性能并不只取决于网络架构，还取决于预训练语料、输入表示和迁移策略[6][18]。

不过，CNN 的局限也比较明显。卷积核天然偏向局部邻域建模，长程依赖需要通过堆叠层数间接获得；同时，频谱图的时间轴和频率轴语义不同，普通二维卷积容易把二者当成类似图像空间维度处理。对于持续时间较长、由多个片段共同决定的声音事件，单纯 CNN 可能难以直接建模远距离片段之间的关系。这也是频谱图 Transformer 和 CNN-Transformer 混合结构受到关注的原因[7][9]。

## 4. 频谱图 Transformer 的发展

Vision Transformer 证明，二维图像可以被切分为 patch，并作为 token 输入 Transformer 编码器；这种思路弱化了卷积结构，强调通过自注意力学习全局关系[1]。音频频谱图同样是二维矩阵，因此研究者自然尝试把时间-频率局部区域切成 patch，让模型学习不同声学片段之间的关系。AST 是这一方向的代表工作，它直接把 Log-Mel Spectrogram 切分为 patch，使用类似 ViT 的 Transformer 编码器进行音频分类，并在 AudioSet、ESC-50 和 Speech Commands 等任务上报告了强表现[7]。

AST 的贡献不只是引入新模型，还提出了一个重要问题：音频分类是否必须依赖卷积结构。AST 的结果说明，经过适当预训练和微调后，纯注意力模型也可以从频谱图中学习有效表示[7]。但 AST 也暴露出音频 Transformer 的现实难点：模型通常需要较多数据或预训练支持，token 数量会影响显存和训练时间，固定输入长度会带来裁剪、填充和位置编码问题[7][8]。

围绕这些问题，PaSST 和 HTS-AT 从效率和结构层面改进音频 Transformer。PaSST 提出 Patchout，在训练中随机丢弃部分频谱 patch，以降低计算成本并增强泛化能力[8]。HTS-AT 使用层次化结构和 token-semantic 模块，试图减少模型参数与训练时间，同时兼顾声音分类和事件定位[9]。这些研究表明，音频 Transformer 的发展并不是简单照搬 ViT，而是在逐渐适应音频任务的时间尺度、频率结构、计算效率和弱定位需求[7][8][9]。

从综合角度看，CNN 与 Transformer 不是简单替代关系。CNN 更强于局部时频纹理建模和小样本稳定训练；Transformer 更适合全局依赖、预训练迁移和多片段关系建模。实际研究中，二者可以形成互补：CNN 可作为低成本基线或局部前端，Transformer 可作为全局建模和迁移学习核心[6][7][9]。

## 5. 自监督音频表征与预训练范式

大规模人工标注音频成本较高，因此自监督学习成为音频分类的重要方向。SSAST 在 AST 基础上引入 masked spectrogram patch modeling，用无标签音频进行生成式和判别式预训练，以减少对大规模有标签数据的依赖[10]。Audio-MAE 进一步借鉴 Masked Autoencoder 思想，高比例遮蔽频谱 patch，并通过重建任务学习通用音频表示；该工作在多个音频和语音分类任务上展示了自监督频谱图预训练的潜力[11]。BYOL-A 则从非对比自监督学习出发，通过增强视图之间的表征预测学习通用音频表示，为不依赖负样本的音频自监督学习提供了另一条路线[12]。

这些工作共同说明，声音事件分类中的关键瓶颈之一是标注效率。完全监督训练需要大量标注数据，而自监督预训练可以利用无标签音频学习底层声学结构和部分语义信息[10][11][12]。不过，自监督方法也存在局限。重建频谱图可能偏向低层声学细节，不一定直接对应分类语义；对比式或预测式方法高度依赖数据增强设计；大规模预训练仍需要较高算力和较长训练时间[11][12]。因此，在声音事件分类中，自监督模型更常作为预训练表示或迁移学习来源，而不是每个下游任务都从头复现完整预训练流程[10][11]。

预训练范式还改变了研究问题的重心。早期研究主要比较手工特征和模型结构；PANNs、AST、SSAST 和 Audio-MAE 之后，研究者更加关注预训练数据来源、标签质量、迁移策略、冻结与微调方式，以及预训练任务和下游分类语义之间的匹配程度[6][7][10][11]。这也是当前声音事件分类从“模型结构竞争”转向“结构、数据和训练策略共同优化”的重要趋势。

## 6. 数据增强、采样与训练策略

数据增强是声音事件分类中提高泛化能力的重要手段。SpecAugment 最初用于语音识别，通过时间遮蔽和频率遮蔽增强模型对局部缺失的鲁棒性[13]；该思想也常被迁移到 Log-Mel Spectrogram 分类中，用于模拟频带缺失、时间片段缺失或局部噪声扰动。mixup 通过对样本和标签进行线性混合，鼓励模型学习更平滑的决策边界，对小样本和类别不平衡任务尤其有价值[14]。PaSST 的 Patchout 则可以看作面向音频 Transformer 的结构化随机丢弃策略，既降低训练成本，也减少模型对局部 patch 的过拟合[8]。

PSLA 的研究进一步说明，音频分类性能并不只取决于网络本身。该工作系统组合了预训练、采样、标签增强、数据增强和模型集成等策略，在 AudioSet 和 FSD50K 上取得较强表现[18]。它提醒研究者：在弱标签和类别不均衡场景中，训练管线的设计往往和模型架构同样重要。对声音事件分类而言，合理的采样策略可以缓解高频类别主导训练的问题；标签增强可以改善弱标签数据的监督质量；数据增强可以减轻模型对特定背景噪声、录音设备或音频长度的依赖[13][14][18]。

因此，数据增强与训练策略应被视为声音事件分类研究的重要组成部分，而不是附属技巧。尤其在小规模数据集或课程项目这类资源有限场景中，预训练、冻结部分层、SpecAugment、mixup、类别均衡采样和早停等方法，往往比盲目扩大模型规模更能稳定提升结果[8][13][14][18]。

## 7. 音视频学习与跨模态迁移

声音事件常与视觉场景存在自然对应关系，例如犬吠与动物画面、汽车鸣笛与交通场景、乐器声与演奏动作。VGGSound 通过大规模音视频片段建立声音事件与视觉内容之间的关联，为音视频事件识别和跨模态学习提供了重要数据基础[16]。VATT 则从视频、音频和文本三种模态中学习 Transformer 表征，说明跨模态自监督学习可以利用不同模态之间的语义对齐改善下游任务[15]。

音视频学习对声音事件分类的价值在于，它提供了纯音频之外的语义先验。当两个声音类别在频谱上相似时，视觉上下文可能帮助区分事件来源；当音频标注不足时，视频或文本模态也可能提供弱监督信号[15][16]。不过，这类方法的工程复杂度明显高于纯音频方法。它需要视频下载、帧抽取、音视频同步、多模态编码和融合策略，同时在线视频数据还存在样本失效、版权和地域可用性问题[16]。

因此，音视频迁移更适合作为声音事件分类的扩展研究方向。已有研究证明跨模态表征具有潜力，但在资源有限的实验中，直接构建完整音视频训练流程可能会分散主要研究目标。更现实的做法是使用已有音视频预训练模型或把音视频学习作为对纯音频方法局限性的讨论与未来工作[15][16]。

## 8. 综合分析与研究空白

综合已有研究，声音事件分类的发展可以概括为三条主线。第一，输入表示从手工特征转向可学习的时频表示。Log-Mel Spectrogram 既保留声学结构，又便于与 CNN 和 Transformer 结合，因此仍是许多工作的核心输入[3][6][7]。第二，模型结构从局部卷积扩展到全局注意力。CNN 在局部纹理和稳定训练上有优势，AST、PaSST 和 HTS-AT 则推动了频谱图 patch 化、全局依赖建模和高效 Transformer 训练[7][8][9]。第三，训练范式从完全监督转向预训练、自监督和训练管线优化。PANNs、SSAST、Audio-MAE、BYOL-A 和 PSLA 都显示，数据来源、标签质量和增强策略是提升音频分类性能的关键[6][10][11][12][18]。

该领域仍存在若干研究空白。首先，小样本数据上的大型 Transformer 泛化仍不稳定，如何在预训练、冻结策略和数据增强之间取得平衡，需要更多系统比较[7][8][13]。其次，多标签声音事件分类受标签噪声、类别不均衡和类别共现影响，不能只依靠整体 Accuracy 判断模型表现，应结合 mAP、F1 和类别级错误分析[5][18]。第三，频谱图虽然形式上类似图像，但时间轴和频率轴具有不同语义，直接迁移图像 ViT 可能忽略音频特有的时间尺度和频率结构[1][7][9]。第四，音视频迁移具有语义优势，但数据获取和训练成本高，如何在纯音频流程和多模态增强之间建立可控、可复现的实验设计仍是挑战[15][16]。

从研究趋势看，未来声音事件分类不太可能只依赖单一模型结构。更有前景的方向是把稳定的时频表示、预训练音频模型、任务适配的 Transformer 结构、数据增强、标签处理和错误分析结合起来。也就是说，模型性能提升来自“表示、结构、数据、训练策略和评价体系”的共同改进，而不是单纯更换一个更大的网络。

## 9. 对后续研究设计的启示

已有研究为声音事件分类后续实验提供了清晰启示。首先，Log-Mel Spectrogram 仍是可靠输入，它既支持 CNN 基线，也支持 AST 类 Transformer 方法[3][7]。其次，纯 Transformer 对预训练和数据量更敏感，因此在有限数据场景中应优先考虑已有预训练模型，而不是从头训练大型模型[7][10][11]。再次，数据增强和训练管线需要被纳入正式实验比较，SpecAugment、mixup、Patchout、类别均衡采样和标签增强都可能显著影响结果[8][13][14][18]。最后，如果后续探索音视频迁移，应明确其作为扩展研究的定位，并优先利用已有预训练表征，以降低数据下载和多模态处理成本[15][16]。

## 10. 结论

本文围绕声音事件分类中的频谱图表示、CNN、音频 Transformer、自监督预训练、数据增强和音视频迁移研究进行了综述。已有文献表明，声音事件分类已经从手工特征和局部卷积建模，发展到以 Log-Mel Spectrogram、预训练音频网络和频谱图 Transformer 为核心的研究阶段。AST 证明了纯注意力模型在音频分类中的可行性，PaSST 和 HTS-AT 进一步改进了效率和结构适配；PANNs、SSAST、Audio-MAE 和 BYOL-A 展示了预训练与自监督学习对标注效率的价值；SpecAugment、mixup、Patchout 和 PSLA 说明训练策略和标签处理对泛化能力同样关键；VATT 与 VGGSound 则为音视频迁移提供了多模态扩展方向。总体来看，声音事件分类的核心挑战已经从单纯选择模型，转向如何在数据规模、标签质量、预训练来源、模型结构和评价指标之间建立可复现、可解释且可扩展的研究方案。
